# Faster CI via parallelism Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cut CI wall-clock by parallelizing the test run with `pytest-xdist` and splitting the single serial CI job into three concurrent jobs (`lint`, `unit`, `e2e`), without changing what is tested.

**Architecture:** Two artifacts change. (1) `pyproject.toml` gains `pytest-xdist` in the existing `dev` dependency group; the lockfile is refreshed. (2) `.github/workflows/ci.yml` is rewritten from one `test` job into three jobs with no `needs:` edges, so GitHub runs them concurrently and wall-clock becomes ≈ `max(unit, e2e)`. `unit` runs `pytest -n auto`; `e2e` runs `pytest -m e2e -n 2`.

**Tech Stack:** GitHub Actions, `uv` (astral), pytest + pytest-django + pytest-xdist, Playwright (chromium), PostgreSQL 16 service.

## Global Constraints

Copied verbatim from the spec; every task's requirements implicitly include these.

- `pytest-xdist` goes in the **existing** `[dependency-groups] dev` group (alongside `pytest`), so bare `uv sync` installs it — a separate non-default group would make `-n auto` fail with "unrecognized arguments: -n".
- `addopts = "-q -m 'not e2e'"` in `pyproject.toml` **must stay**. `unit` inherits `-m 'not e2e'` from it; the `e2e` job's explicit command-line `-m e2e` intentionally overrides it (last `-m` wins). Do not "tidy" either marker away.
- Every job sets `runs-on: ubuntu-latest` and runs the shared prelude before its steps: `actions/checkout@v5`, then `astral-sh/setup-uv@v6` with `python-version: "3.13"` and `enable-cache: true`, then `uv sync`.
- All commands keep their `uv run` (and `python -m` / `python manage.py`) invocation form.
- `unit` runs `migrate` and `setup_roles` **before** pytest (fail-fast; they touch only the `libli` DB, not the per-worker `test_libli_*` DBs, so order is correctness-neutral).
- Each DB-bound job (`unit`, `e2e`) carries its own Postgres `services` block and its own `env` block (`DJANGO_SETTINGS_MODULE`, `DATABASE_URL`, `DJANGO_SECRET_KEY`). `lint` carries neither.
- The Postgres service provisions role **and** database `libli` (`POSTGRES_USER: libli`, `POSTGRES_DB: libli`); the `libli` role is the image superuser, so it has CREATEDB for the per-worker test databases.
- The `e2e` Playwright browser cache uses `actions/cache` keyed on the resolved Playwright version; the version-derivation step must run **before** the cache step.
- The top-level `concurrency: { group: ci-${{ github.ref }}, cancel-in-progress: true }` block and the `on: { push: [master], pull_request: {} }` trigger are preserved verbatim.
- Per-job `timeout-minutes`: indicatively ~5 (`lint`), ~15 (`unit`), ~20 (`e2e`).
- **Do NOT run the full e2e suite locally during execution** (background `-m e2e` spawns runaway headless browsers). e2e parallel-safety under `-n 2` is validated by CI, not locally.
- Authoritative DoD: a green GitHub Actions run, which (given the `push: [master]` + `pull_request` trigger) first reports on this branch via its **open PR**.

### Task 1: Add pytest-xdist and prove the unit suite is parallel-safe under `-n auto`

**Files:**
- Modify: `pyproject.toml` (the `[dependency-groups] dev` array)
- Modify: `uv.lock` (regenerated by `uv lock`)
- Possibly modify (only if the `-n auto` run surfaces a parallel-safety failure): `tests/conftest.py` or `config/settings/test.py`

**Interfaces:**
- Consumes: nothing (first task).
- Produces: `pytest-xdist` available in the `dev` group so `uv run python -m pytest -n auto` is a recognized invocation for Task 2's workflow and for CI.

- [ ] **Step 1: Add `pytest-xdist` to the `dev` dependency group**

Edit `pyproject.toml`. In `[dependency-groups]`, add `pytest-xdist` to the `dev` array (keep the array alphabetical to match the existing order):

```toml
[dependency-groups]
dev = [
    "factory-boy>=3.3.3",
    "pytest>=9.0.3",
    "pytest-django>=4.12.0",
    "pytest-playwright>=0.5.0",
    "pytest-xdist>=3.6.0",
    "ruff>=0.15.17",
]
```

Note: `>=3.6.0` is a floor; `uv lock` in Step 2 resolves to the newest version compatible with `pytest>=9.0.3`. If resolution fails because a too-old cap is chosen, raise the floor (e.g. `>=3.8.0`) and re-run.

- [ ] **Step 2: Refresh the lockfile and sync**

Run:
```bash
uv lock
uv sync
```
Expected: `uv lock` reports `pytest-xdist` (and its `execnet` dependency) added; `uv sync` installs them into the venv with no error.

- [ ] **Step 3: Verify `-n` is now a recognized pytest argument**

Run:
```bash
uv run python -m pytest -n auto --collect-only -q
```
Expected: pytest collects tests and exits 0. It must NOT print `error: unrecognized arguments: -n` (which is the failure mode when xdist is absent or in a non-default group). The collected set excludes e2e (via the `-m 'not e2e'` in `addopts`).

- [ ] **Step 4: Run the full unit suite under `-n auto` — the parallel-safety gate**

Run (foreground; this is the risk-retirement step — needs the local Postgres at `postgres://libli:libli@localhost:5432/libli`):
```bash
uv run python -m pytest -n auto
```
Expected: the entire unit suite passes, distributed across workers. This proves xdist parallel-safety.

If — and only if — the run surfaces a parallel-safety failure, fix it per the spec's three failure modes before committing, then re-run Step 4 until green:
- **Filesystem collision** on a shared global root (`MEDIA_ROOT` / `TRANSFER_STAGING_DIR`): the existing tests already redirect these to per-test `tmp_path`, so this is not expected; if a new collision appears, give the offending test a per-test `tmp_path` root, or add a per-worker root in `config/settings/test.py` keyed on `os.environ.get("PYTEST_XDIST_WORKER", "gw-single")`.
- **Ordering assumption** exposed by reordering: co-locate the interdependent tests with `@pytest.mark.xdist_group("<name>")` (keeps them on one worker in order), rather than a non-existent "serial" marker.
- **Concurrent `CREATE DATABASE test_libli_gwN` race** at session start (e.g. "template1 is being accessed by other users"): re-run; if persistent, this is a bootstrap race, not a test bug — note it for CI and prefer `--create-db`/reuse.

- [ ] **Step 5: Confirm ruff is still clean (the changed file is TOML, but keep the tree green)**

Run:
```bash
uv run ruff check .
uv run ruff format --check .
```
Expected: both pass (no Python changed unless a Step-4 fix was needed; if it was, ensure it is formatted).

- [ ] **Step 6: Commit**

```bash
git add pyproject.toml uv.lock
# include tests/conftest.py or config/settings/test.py ONLY if Step 4 required a fix
git commit -m "build(ci): add pytest-xdist; unit suite green under -n auto"
```

### Task 2: Rewrite `.github/workflows/ci.yml` into three concurrent jobs

**Files:**
- Modify (full rewrite): `.github/workflows/ci.yml`

**Interfaces:**
- Consumes: `pytest-xdist` from Task 1 (so `-n auto` / `-n 2` are valid).
- Produces: the final CI workflow. No later task depends on it.

- [ ] **Step 1: Replace the entire workflow file**

Overwrite `.github/workflows/ci.yml` with exactly this content:

```yaml
name: ci
on:
  push:
    branches: [master]
  pull_request:
concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true
jobs:
  lint:
    runs-on: ubuntu-latest
    timeout-minutes: 5
    steps:
      - uses: actions/checkout@v5
      - uses: astral-sh/setup-uv@v6
        with:
          python-version: "3.13"
          enable-cache: true
      - run: uv sync
      - run: uv run ruff check .
      - run: uv run ruff format --check .
  unit:
    runs-on: ubuntu-latest
    timeout-minutes: 15
    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_USER: libli
          POSTGRES_PASSWORD: libli
          POSTGRES_DB: libli
        ports: ["5432:5432"]
        options: >-
          --health-cmd pg_isready --health-interval 10s
          --health-timeout 5s --health-retries 5
    env:
      DJANGO_SETTINGS_MODULE: config.settings.test
      DATABASE_URL: postgres://libli:libli@localhost:5432/libli
      DJANGO_SECRET_KEY: ci-secret
    steps:
      - uses: actions/checkout@v5
      - uses: astral-sh/setup-uv@v6
        with:
          python-version: "3.13"
          enable-cache: true
      - run: uv sync
      - run: uv run python manage.py migrate        # verify the real deploy path (before tests, fail fast)
      - run: uv run python manage.py setup_roles     # asserts seed_roles works post-migrate
      - run: uv run python -m pytest -n auto
  e2e:
    runs-on: ubuntu-latest
    timeout-minutes: 20
    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_USER: libli
          POSTGRES_PASSWORD: libli
          POSTGRES_DB: libli
        ports: ["5432:5432"]
        options: >-
          --health-cmd pg_isready --health-interval 10s
          --health-timeout 5s --health-retries 5
    env:
      DJANGO_SETTINGS_MODULE: config.settings.test
      DATABASE_URL: postgres://libli:libli@localhost:5432/libli
      DJANGO_SECRET_KEY: ci-secret
    steps:
      - uses: actions/checkout@v5
      - uses: astral-sh/setup-uv@v6
        with:
          python-version: "3.13"
          enable-cache: true
      - run: uv sync
      - name: Resolve Playwright version
        run: echo "PLAYWRIGHT_VERSION=$(uv run playwright --version | awk '{print $2}')" >> "$GITHUB_ENV"
      - name: Cache Playwright browsers
        uses: actions/cache@v4
        with:
          path: ~/.cache/ms-playwright
          key: ms-playwright-${{ runner.os }}-${{ env.PLAYWRIGHT_VERSION }}
      - run: uv run playwright install --with-deps chromium
      - run: uv run python -m pytest -m e2e -n 2
```

- [ ] **Step 2: Verify the workflow is well-formed YAML**

Run:
```bash
uv run python -c "import yaml, pathlib; d = yaml.safe_load(pathlib.Path('.github/workflows/ci.yml').read_text()); print(sorted(d['jobs'])); assert sorted(d['jobs']) == ['e2e', 'lint', 'unit']; assert 'needs' not in d['jobs']['e2e'] and 'needs' not in d['jobs']['unit'] and 'needs' not in d['jobs']['lint']; print('OK: 3 concurrent jobs, no needs edges')"
```
Expected: prints `['e2e', 'lint', 'unit']` then `OK: 3 concurrent jobs, no needs edges`.

- [ ] **Step 3: Manually trace the Global Constraints against the file**

Confirm by reading `.github/workflows/ci.yml`:
- `concurrency` block and `on:` trigger are byte-for-byte the originals.
- `unit` has `migrate` + `setup_roles` **before** `pytest -n auto`.
- `e2e` has the version-derivation step **before** the `actions/cache` step, and runs `pytest -m e2e -n 2`.
- `lint` has no `services:` and no `env:`.
- each job has `runs-on: ubuntu-latest` and a `timeout-minutes`.

No command output to assert here; it is a read-and-verify step.

- [ ] **Step 4: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: split into concurrent lint/unit/e2e jobs with xdist parallelism"
```
